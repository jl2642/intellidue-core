# IntelliDue Operating Model

## 1. System role

IntelliDue is a ChatGPT-led due-diligence production system with a public, project-neutral control core.

The professional analysis engine is the authorized ChatGPT workspace operating on private project sources. IntelliDue Core does not replace professional judgment and does not independently perform due-diligence analysis. It supplies reusable contracts, validators, manifests, release controls, private-runtime promotion and non-regression gates so that different projects follow the same quality and privacy discipline.

## 2. Responsibilities

### Project owner / user

The project owner:

- authorizes the project identity, purpose, scope, confidentiality and audience;
- provides or connects the private source rooms;
- identifies material commercial context that is not contained in the documents;
- approves investigation priorities, budget, timing and decision gates;
- supplies named finance, legal, engineering/HSE or other professional reviewers when formal reliance is required;
- decides whether a preliminary product may be used for negotiation, whether additional evidence must be obtained, and whether a final investment decision may be considered.

### ChatGPT private project workspace

The ChatGPT workspace:

- inventories, deduplicates and classifies source material;
- defines Questions, Evidence, conflicts, critical gates and blocked outputs;
- performs document analysis, calculations, cross-checking, web research when authorized and professional synthesis;
- produces workpapers, specialist reports, integrated reports, management products and bespoke follow-up deliverables;
- preserves uncertainty and does not promote unsupported facts, valuation outputs or decision readiness;
- runs content, calculation, citation, consistency, publication and clean-chat acceptance;
- maps incremental evidence only to affected products and preserves prior accepted releases.

### IntelliDue Core / GitHub

The public GitHub repository:

- freezes project-neutral schemas, contracts, runbooks and quality gates;
- validates controlled state, release lock, package validation and product manifests;
- builds deterministic packages and reconciles every file by path, size and SHA-256;
- maintains immutable releases and Current, Last-success and Archive pointers;
- enforces project-bound private-runtime promotion, rollback, crash recovery and contamination rejection;
- provides synthetic regression tests, CI, security scanning and release automation;
- never acts as the source room, analysis engine or storage location for a real project's files, reports, filenames, hashes or business facts.

### Private storage

Private File Library and approved cloud data rooms hold real project sources and private products. They are separate from the public repository. Access control, sharing permissions, retention and deletion remain the responsibility of the storage owner and platform administrator.

## 3. Product authority

Authority flows upward through the product stack:

1. Source Registry and immutable intake snapshot;
2. Evidence Register, Questions, conflicts and critical gates;
3. professional workpapers and recalculations;
4. specialist reports;
5. integrated due-diligence report;
6. stage-gate or investment-committee memo.

A management product may compress approved parents, but it may not create a new fact or stronger professional conclusion. An integrated report may not upgrade the weakest material workpaper. Final Full DD is an evidence and professional-signoff upgrade, not a writing upgrade.

## 4. Completion meaning

Operational release of IntelliDue Core means that the reusable system has passed repository governance, public CI, private-project replay and release acceptance. It does not mean every future project is automatically complete, nor that every preliminary Reader Pack is a Final Full DD product. Each project remains subject to its own evidence gates, professional procedures and decision authority.
