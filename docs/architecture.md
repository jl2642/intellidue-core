# IntelliDue Production Baseline v1.0.0

Freeze date: 2026-07-22
Status: PRODUCTION_BASELINE_FROZEN_READY_FOR_PUBLIC_GITHUB_CORE_DEPLOYMENT_AND_PRIVATE_PROJECT_OPERATION

## 1. Frozen System Scope

The frozen system supports a complete **preliminary / red-flag due-diligence production workflow**:

1. dual-repository project intake and source identity control;
2. file inventory, duplicate control, disposition and review-depth disclosure;
3. Question / Evidence / critical-gate control;
4. professional workpaper and recalculation baseline;
5. specialist reports by material domain;
6. integrated issue-led due-diligence report;
7. stage-gate investment committee decision memo;
8. Reader / Control package separation;
9. Current / Archive / Last-success pointers;
10. package-only clean-room recovery and non-regression acceptance.

## 2. Frozen Product Classes

- `PRELIMINARY_RED_FLAG_DD`: supported and Production Golden.
- `INTEGRATED_RED_FLAG_DD`: supported when professional parents and source limits are disclosed.
- `STAGE_GATE_IC_MEMO`: supported for controlled evidence-closure or negotiation authorization.
- `FINAL_FULL_DD`: not automatically authorized; requires project evidence closure and complete professional procedures.
- `FINAL_INVESTMENT_APPROVAL`: not automatically authorized; requires decision-ready financial, debt, valuation, legal and technical inputs.

## 3. Non-negotiable Gates

- no file-registration count may be described as full professional review;
- no diagnostic value may be promoted to a transaction-ready output without evidence;
- unresolved conflicts must remain visible;
- Reader products may not contain internal phase, test, Question or Source IDs;
- formal reliance requires named human reviewers;
- public GitHub assets may not contain private project data, reports, filenames, hashes or commercial facts.

## 4. Pilot Acceptance Boundary

A real-project pilot may prove that the system can produce a stable preliminary/red-flag Reader Pack in layers. A pilot acceptance must never be interpreted as proof that the underlying project reached Final Full DD or final investment readiness. Project-specific facts, names, counts and conclusions remain private.
